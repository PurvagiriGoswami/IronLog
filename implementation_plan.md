# IronLog — Phase 1: Local-First Core Implementation Plan

Build a **production-ready, offline-first workout tracking PWA** using Next.js 14+, TypeScript, Tailwind CSS, shadcn/ui, Dexie.js (IndexedDB), and Recharts. Phase 1 delivers a fully functional app with zero internet dependency.

## User Review Required

> [!IMPORTANT]
> **Tailwind CSS version**: The latest `create-next-app` defaults to Tailwind v4. shadcn/ui has full support for Tailwind v4. I'll proceed with the default (v4) unless you prefer v3.

> [!IMPORTANT]
> **Exercise data**: I'll generate the full 200+ exercise JSON with all fields (muscles, equipment, form cues, common mistakes, default sets/reps). This is a large data file (~3000 lines). I'll populate it with real, accurate exercise data — not placeholder content.

> [!WARNING]
> **Scope reality check**: Phase 1 alone is a very large application. Even at aggressive speed, this will produce 50-80+ files and 15,000+ lines of code. I'll build it component-by-component so you can review and test incrementally. The full build will take multiple interactions.

## Open Questions

> [!IMPORTANT]
> **App name confirmation**: The spec uses "IronLog" in the PWA manifest. Should I use this name throughout, or do you have a different preference?

> [!IMPORTANT]
> **Deployment target**: Are you planning to deploy this anywhere (Vercel, self-hosted) during Phase 1, or purely local `npm run dev` for now? This affects whether I set up environment variable handling for production.

## Verification Plan

### Automated Tests
- `npm run build` — ensure zero TypeScript errors and successful production build
- `npm run lint` — ensure no ESLint errors
- Manual browser testing of all core flows

### Manual Verification (browser testing flow)
1. **First launch**: onboarding flow renders, equipment selection works, settings persist
2. **Exercise library**: search works, filters work, favorites toggle, custom exercise creation
3. **Workout logging**: start workout → add exercise → log sets → complete set → rest timer → PR detection → finish workout → summary screen
4. **Templates**: save completed workout as template → start from template → verify pre-populated data
5. **History**: past workouts appear → detail view → edit past session → delete session
6. **Progress**: dashboard cards show correct data → charts render → heatmap → body map SVG → exercise analytics
7. **Program builder**: create program → set weekly schedule → add exercises to days → activate → start today's session
8. **Utility tools**: plate calculator → 1RM calculator → warm-up generator → data export
9. **PWA**: installable on mobile Chrome, works offline, service worker caching active
10. **Responsive design**: test on 375px (iPhone SE), 390px (iPhone 14), 768px (iPad), 1440px (desktop)

### Build Order
I'll build components in this order, ensuring each is testable before moving to the next:
1. Scaffold + Design System (Components 1-2)
2. Database + Exercise Data (Components 3-4)
3. App Layout + Navigation (Component 11)
4. Onboarding (Component 10)
5. Workout Logging (Component 5) — **the core feature**
6. Templates & History (Component 6)
7. Progress & Analytics (Component 7)
8. Program Builder (Component 8)
9. Utility Tools (Component 9)
10. Profile & Settings (Component 12)
